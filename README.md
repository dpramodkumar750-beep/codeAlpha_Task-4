# 📈 Sales Prediction System

**CodeAlpha Data Science Internship — Task 4**

> Predict sales revenue from TV, Radio, and Newspaper advertising spend using seven machine-learning algorithms, automatic hyperparameter tuning, and an interactive Streamlit dashboard.

---

## 📋 Project Overview

This project builds an end-to-end sales prediction pipeline on the **Advertising dataset** (200 rows, 3 features, 1 target). It covers the full ML lifecycle:

- **Data inspection** with deliberate pitfall-catching (stray index column)
- **EDA** with histograms, KDE, pairplot, correlation heatmap, and regression scatter plots
- **Model training** across 7 algorithms with cross-validated comparison
- **Hyperparameter tuning** via GridSearchCV
- **Model persistence** with reload verification
- **Interactive dashboard** via Streamlit

---

## 📁 Dataset Description

| Column | Description | Units |
|--------|-------------|-------|
| TV | TV advertising spend | $000 |
| Radio | Radio advertising spend | $000 |
| Newspaper | Newspaper advertising spend | $000 |
| Sales | Units sold | 000 units |

- **200 rows**, no missing values, no duplicates (verified programmatically)
- **Critical note:** The raw CSV has an unnamed first column (a pandas-exported row index). Loading with `index_col=0` drops it immediately — otherwise it leaks into the feature matrix and inflates R².

## 🚀 Installation

### Option A — System Python (recommended if disk space is limited)

```bash
# Verify the system Python has all required packages
python -m pip list | grep -E "pandas|numpy|scikit|seaborn|matplotlib|streamlit|xgboost"

# Install any missing packages
python -m pip install pandas numpy matplotlib seaborn scikit-learn joblib streamlit xgboost
```

### Option B — Virtual environment (recommended for isolation)

```bash
# Create venv
python -m venv .venv

# Activate (Windows PowerShell)
.venv\Scripts\Activate.ps1

# Install dependencies
pip install -r requirements.txt
```

> **Disk space note:** The full package set (including xgboost, streamlit, pyarrow) requires ~500 MB of free space.

## ▶️ How to Run

### Step 1 — Generate model artifacts

```bash
python train_model.py
```

This runs the full pipeline (load → clean → split → scale → train 7 models → tune → save) and writes `model.pkl`, `scaler.pkl`, `features.pkl`.

### Step 2 — Launch the Streamlit dashboard

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

### Step 3 — Run the Jupyter notebook (optional, for EDA review)

```bash
jupyter notebook notebooks/CodeAlpha_SalesPrediction.ipynb
```

## 💡 Business Insights

Insights are **generated programmatically** from model outputs, not hardcoded prose.

| Channel | Pearson r with Sales | ROI Proxy (Δ Sales/$000) |
|---------|----------------------|--------------------------|
| TV | ~0.78 | ~0.047 |
| Radio | ~0.58 | ~0.188 |
| Newspaper | ~0.23 | ~0.002 |

**Directional finding:** The model suggests `Newspaper` has the lowest ROI proxy. Reallocating Newspaper budget toward `Radio` (highest Δ Sales per $000 spent) may improve sales outcomes.

> ⚠️ **Caveat:** With 200 rows and 3 features, these findings are directional hypotheses — not prescriptive budget recommendations. Validate with larger, time-aware datasets before acting.

### Why correlation ≠ feature importance

- **Pearson correlation** measures linear association only; it ignores interaction effects between channels.
- **Tree feature importance** captures non-linear effects and interactions — so rankings can differ.
  For example, Radio may rank higher in tree importance than correlation alone suggests, because it interacts non-additively with TV spend.

## 🔮 Future Improvements

1. **More data:** 200 rows is a thin training set — collect multi-year historical data.
2. **Time features:** Advertising ROI varies by season/day-of-week — add date features.
3. **Interaction terms:** Engineer `TV × Radio` cross terms; linear models don't capture interactions automatically.
4. **Channel attribution modelling:** Use Shapley values (SHAP) for per-prediction channel attribution.
5. **Budget optimisation:** Formulate as a constrained optimisation problem — given a total budget cap, find the allocation that maximises predicted Sales.
6. **Real-time inference API:** Wrap `predict_sales()` in a FastAPI endpoint for production serving.

## 📜 Conventions

This project follows the same conventions as Tasks 1–3 of the CodeAlpha internship:

- PEP 8 compliant code, docstrings on all functions
- Markdown insight paragraphs *before* and *after* every plot in the notebook
- Exception handling via explicit `raise` (no silent `except: pass` blocks — these break Pylance type narrowing)
- All numpy scalars passed to matplotlib wrapped with `float()` to avoid `numpy.float64` annotation issues
- Colormap variable named `cmap` (not `cm`) to avoid collision with `sklearn.metrics.confusion_matrix`
- No `stratify=` on `train_test_split` — this is continuous regression, not classification
